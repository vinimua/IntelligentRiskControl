# 1. 通用约定

## 1.1 Base URL

```text
开发环境：http://localhost:8000
测试环境：https://test-modelops.example.com
生产环境：https://modelops.example.com
```

## 1.2 请求头

| 请求头 | 必填 | 说明 |
|---|---|---|
| `Content-Type` | 是 | `application/json` |
| `Authorization` | 是 | `Bearer {access_token}` |
| `X-Trace-Id` | 是 | 端到端链路 ID，UUID v4 |
| `X-Schema-Version` | 是 | 接口 Schema 版本，如 `1.0` |

## 1.3 通用响应

### 成功响应

```json
{
  "success": true,
  "code": "OK",
  "message": "success",
  "data": {},
  "trace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

`data` 为具体接口的返回体，可为对象、数组或 `null`。

### 错误响应

```json
{
  "success": false,
  "code": "ERROR_CODE",
  "message": "human readable error description",
  "data": null,
  "trace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "retryable": false,
  "details": {}
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `success` | bool | 固定 `false` |
| `code` | string | 稳定错误码 |
| `message` | string | 人类可读描述 |
| `data` | null | 固定 `null` |
| `trace_id` | string | 与请求一致 |
| `retryable` | bool | 重试是否安全 |
| `details` | object | 结构化补充信息 |

## 1.4 分页约定

请求：

```json
{
  "page": 1,
  "page_size": 20
}
```

| 参数 | 类型 | 默认值 | 最大值 | 说明 |
|---|---|---|---|
| `page` | integer | 1 | — | 从 1 开始 |
| `page_size` | integer | 20 | 100 | 每页条数 |

响应：

```json
{
  "items": [],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 156,
    "total_pages": 8
  }
}
```

## 1.5 时间格式

所有时间字段使用 ISO 8601 UTC：

```text
2026-07-14T02:00:00Z
```

## 1.6 排序约定

```json
{
  "sort_by": "created_at",
  "sort_order": "desc"
}
```

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `sort_by` | string | 接口定义 | 排序字段 |
| `sort_order` | string | `desc` | `asc` 或 `desc` |

## 1.7 权限角色

| 角色 | 权限范围 |
|---|---|
| `VIEWER` | 查看监控、诊断和模型报告 |
| `MODELER` | 创建迭代、查看实验，**不能上线** |
| `RISK_REVIEWER` | 审批根因、规则例外和模型准入 |
| `DEPLOYER` | 执行受审批的发布与回滚 |
| `KNOWLEDGE_EDITOR` | 上传文档、维护来源、修正实体关系 |
| `KNOWLEDGE_REVIEWER` | 审核 Schema 扩展、执行器绑定和高风险关系 |
| `KNOWLEDGE_PUBLISHER` | 应用/回滚权重版本、禁用关系、生成快照 |
| `ADMIN` | 配置模型、知识 Schema、阈值和权限 |

## 1.8 统一错误码

| 错误码 | HTTP 状态码 | 说明 | retryable |
|---|---|---|---|
| `OK` | 200 | 成功 | — |
| `BAD_REQUEST` | 400 | 请求参数校验失败 | false |
| `VALIDATION_ERROR` | 422 | Pydantic 校验失败 | false |
| `UNAUTHORIZED` | 401 | 未认证或 Token 过期 | false |
| `FORBIDDEN` | 403 | 无权限 | false |
| `NOT_FOUND` | 404 | 资源不存在 | false |
| `CONFLICT` | 409 | 幂等键冲突或状态冲突 | false |
| `INVALID_ALERT_CODE` | 400 | 未注册告警编码 | false |
| `INVALID_ROOT_CAUSE_CODE` | 400 | 未注册根因编码 | false |
| `INVALID_STRATEGY_CODE` | 400 | 未注册策略编码 | false |
| `INVALID_METHOD_CODE` | 400 | 未注册方法编码 | false |
| `VALIDATION_METHOD_NOT_IMPLEMENTED` | 501 | 方法未绑定执行器 | false |
| `KG_PATH_NOT_FOUND` | 404 | Profile 下无可用图路径 | false |
| `RULE_NOT_FOUND` | 404 | 规则版本不存在 | false |
| `DATASET_POLICY_VIOLATION` | 403 | W4/OOT 窗口违规访问 | false |
| `QDRANT_RETRIEVAL_DEGRADED` | 200 | 文档检索降级（响应中标记） | true |
| `KG_QUERY_ERROR` | 502 | Neo4j 查询失败 | true |
| `CONTEXT_BUILD_ERROR` | 500 | 上下文组装失败 | depends |
| `WORKER_TIMEOUT` | 504 | 异步任务超时 | true |
| `INTERNAL_ERROR` | 500 | 服务内部错误 | true |

# 2. 工作流 API

## 2.1 启动生命周期工作流

```text
POST /api/lifecycle-runs
权限：MODELER / ADMIN
```

**请求体：**

```json
{
  "model_id": "credit_score_model",
  "champion_version": "credit_score_v2",
  "trigger_type": "SCHEDULED_TRIGGER",
  "data_track": "NATURAL",
  "scenario_id": null,
  "monitor_window_id": "W3",
  "baseline_id": "BASELINE_001"
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `model_id` | string | 是 | 稳定模型编码 |
| `champion_version` | string | 是 | 当前 Champion 版本编码 |
| `trigger_type` | string | 是 | `SCHEDULED_TRIGGER` / `THRESHOLD_TRIGGER` / `ABNORMAL_TRIGGER` / `MANUAL_TRIGGER` / `DEPLOYMENT_FAILURE_ANALYSIS` |
| `data_track` | string | 是 | `NATURAL` / `SCENARIO` |
| `scenario_id` | string | 否 | SCENARIO 轨必填，NATURAL 轨必须为 null |
| `monitor_window_id` | string | 否 | 指定监控窗口，不传则自动选择最新 |
| `baseline_id` | string | 否 | 指定健康基准，不传则自动选择 |
| `parent_lifecycle_run_id` | string | 否 | DEPLOYMENT_FAILURE_ANALYSIS 时必须传入 |

**响应体（201 Created）：**

```json
{
  "success": true,
  "code": "OK",
  "message": "lifecycle run created and started",
  "data": {
    "lifecycle_run_id": "LIFE_000001",
    "model_id": "credit_score_model",
    "champion_version": "credit_score_v2",
    "trigger_type": "SCHEDULED_TRIGGER",
    "data_track": "NATURAL",
    "current_phase": "MONITORING",
    "status": "RUNNING",
    "created_at": "2026-07-14T02:00:00Z"
  },
  "trace_id": "TRC_000001"
}
```

**错误场景：**

| 场景 | 错误码 |
|---|---|
| model_id 不存在 | `NOT_FOUND` |
| champion_version 不是 ACTIVE | `BAD_REQUEST` |
| SCENARIO 轨未传 scenario_id | `VALIDATION_ERROR` |
| DEPLOYMENT_FAILURE_ANALYSIS 未传 parent | `VALIDATION_ERROR` |

## 2.4 恢复暂停的工作流

```text
POST /api/lifecycle-runs/{lifecycle_run_id}/resume
权限：RISK_REVIEWER / DEPLOYER / ADMIN
```

**请求体（人工复核场景）：**

```json
{
  "action": "APPROVE",
  "reviewer": "zhangsan",
  "comment": "根因分析合理，同意进入迭代",
  "override_action": null
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `action` | string | 是 | `APPROVE` / `REJECT` / `OVERRIDE` |
| `reviewer` | string | 是 | 审批人标识 |
| `comment` | string | 是 | 审批意见 |
| `override_action` | string | 否 | `OVERRIDE` 时指定强制动作 |

**响应体（200 OK）：**

```json
{
  "success": true,
  "code": "OK",
  "message": "workflow resumed",
  "data": {
    "lifecycle_run_id": "LIFE_000001",
    "new_phase": "ITERATING",
    "status": "RUNNING"
  },
  "trace_id": "TRC_000004"
}
```

**错误场景：**

| 场景 | 错误码 |
|---|---|
| 工作流未处于暂停状态 | `CONFLICT` |
| 用户无审批权限 | `FORBIDDEN` |
